I have experience using Git for version control in Cloud and DevOps projects. I regularly work with repositories, branches, commits, pull requests, merge conflicts, GitHub, and CI/CD pipelines. ---

### Q2. What is Git?

<details>
<summary><b>✅ Show Answer</b></summary>

Git is a Distributed Version Control System (DVCS) that tracks source code changes, maintains project history, and enables multiple developers to collaborate efficiently.

</details>

---

### Q3. What is GitHub?

<details>
<summary><b>✅ Show Answer</b></summary>

GitHub is a cloud-based platform used to host Git repositories and provide collaboration features such as Pull Requests, Issues, Actions, and Code Reviews.

</details>

---

### Q4. Difference between Git and GitHub?

<details>
<summary><b>✅ Show Answer</b></summary>

Git is the version control tool, while GitHub is a platform that hosts Git repositories and adds collaboration features.

</details>

---

### Q5. Explain your Git workflow.

<details>
<summary><b>✅ Show Answer</b></summary>

Clone Repository → Create Feature Branch → Make Changes → Commit → Push → Pull Request → Code Review → Merge → Deploy.

</details>

# 📝 Section 10 — 5-Minute Final Revision

Remember these concepts before entering the interview:

- Git is a DVCS.
- Every Commit creates a snapshot.
- HEAD points to the current commit.
- Staging Area prepares changes for Commit.
- Clone creates a local copy of a repository.
- Fetch downloads changes without merging.
- Pull = Fetch + Merge.
- Merge combines branch history.
- Rebase rewrites commit history.
- Revert safely undoes shared commits.
- Reset changes local history.
- Cherry-pick copies specific commits.
- Reflog helps recover lost commits.
- Tags mark releases.
- Pull Requests enable code review.
- Protected Branches secure production.
- Git Hooks automate tasks.
- Git LFS manages large files.
- GitFlow and Trunk-Based Development are common branching strategies.
- GitOps uses Git as the single source of truth.
- Always follow branch naming, commit message, and repository governance standards.
